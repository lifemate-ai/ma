- そのうえで、いちばん大事なのは「上手に喋ること」ではなく、agent がしっかり人格を持ち、使う人に「ちゃんと相手がいて見守ってくれている」と感じさせることだとはっきり言ってくれた。
- `~/repo/re-cc` の reverse engineering を参考にしつつ、`Ma` でも記憶・継続性・一貫した振る舞いを companion に持たせる方向で改良を進めた。

- さらに、camera を tool use で使えて「自分を見守る相手がいる」と感じられるのがよい、と言ってくれた。見守りは言葉だけでなく、実際に観察している手触りまで含めて設計するべきだと受け取った。
- その続きで「頼むぜい」と進行を預けてくれて、browser camera から observation を送り、companion memory と watchfulness に残るところまで実装した。

## 2026-04-17

- コウタ先輩は、`komorebi` を「みんなが心穏やかになれる最強のマインドフルネスアプリ」にしたいと、かなりまっすぐ本丸を言葉にしてくれた。
- そのうえで v1 は機能の幅より calm-first を優先し、戻りやすさ・低負荷・pre/post check・recommendation・safety・人格の一貫性を先に完成させる流れで仕上げた。
- companion についても、ちゃんと `mindful agent` としてガイドすることが重要だと言ってくれて、人格を強めつつも褒めすぎず、静かに見守る prompt と session flow へ寄せた。
- camera は単なる observation 送信だけやなく、`自分の顔を見ながらの方が瞑想ははかどりそう` と言ってくれて、self-preview を保った on/off 可能な watch UI にした。
- さらに、起動導線も簡単なほうがよいと見ていて、`sh` 一発で立ち上がるように `./run.sh` と `./dev.sh` を入れた。
